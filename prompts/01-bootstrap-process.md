# Prompt 1 — bootstrap project-local process

Read this repository first, especially:
- `docs/pipeline.md`
- `docs/adaptation-guide.md`
- `docs/operator-runbook.md`

Then inspect the target repo you are actually working in.

Your job:
- create a project-local version of this portable pipeline
- choose repo-appropriate task/backlog/automation roots
- install the canonical process docs into the target repo
- create durable state/template stubs
- do not create live cron jobs yet

Requirements:
- adapt, do not blindly copy names or paths
- keep the invariants from `docs/pipeline.md`
- report exact files created and any unresolved local decisions
